Build a professional full-stack IoT mobile application for a smart RO water purifier retrofit.

PROJECT OVERVIEW
This is a hardware + software integrated system based on an ESP32. The ESP32 reads sensors from an RO purifier, sends data through MQTT, stores data in MongoDB Atlas through a Node.js + Express backend, and the mobile app is built in React Native. The app must look professional, clean, and industry-standard, not flashy. It should be easy to extend later with TinyML and additional features.

CURRENT HARDWARE INPUTS
Use the following hardware pin mapping exactly:

1) Temperature sensor:
- Sensor: DS18B20 waterproof digital temperature sensor
- Data pin: GPIO4
- Red wire -> 3V3
- Black wire -> GND
- Yellow wire -> GPIO4
- 4.7k pull-up resistor between Red (3V3) and Yellow (DATA)

2) Water level sensor:
- Analog signal pin: GPIO35
- + -> 3V3
- - -> GND

3) TDS sensor:
- Analog signal pin: GPIO34
- + -> 3V3
- - -> GND
- Use the analog TDS sensor already discussed in the project

4) Relay:
- IN -> GPIO25
- VCC -> 3V3
- GND -> GND

5) OLED display:
- GND -> GND
- VDD -> 3V3
- SCK/SCL -> GPIO22
- SDA -> GPIO21

PROJECT LOGIC
- The system monitors temperature, TDS, and water level in real time.
- The pump/relay control is based on temperature threshold logic.
- Current threshold:
  - If temperature > 35°C, the pump should turn OFF.
  - If temperature <= 35°C, the pump should turn ON.
- This threshold logic should be clearly visible in the app.
- The app should show live sensor values and relay state.
- The app should also support manual override later, but for now the core implementation should be automatic with clear UI indicators.

MQTT REQUIREMENTS
Use MQTT for real-time communication between ESP32 and backend/app.

Use topics like:
- ro/sensor/temperature
- ro/sensor/tds
- ro/sensor/waterlevel
- ro/device/pump
- ro/device/status
- ro/device/alert

Message format should be JSON. Example:
{
  "temperature": 32.5,
  "tds": 189,
  "waterLevel": 4.2,
  "pumpState": "ON",
  "threshold": 35,
  "alert": "NORMAL"
}
